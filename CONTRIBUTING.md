# Contributing Guidelines for Destiny Tour & Travels

Welcome to the Destiny Tour & Travels project! We appreciate your interest in contributing. Below are the guidelines that we encourage all contributors to follow.

## Code Standards
- Use consistent naming conventions for variables, classes, and functions. 
- Follow the coding style used in the existing codebase.
- Include comments to explain the logic of complex code snippets.
- Utilize ES6 features appropriately.

## Commit Conventions
- Use the imperative mood in commit messages: "Add feature" instead of "Added feature".
- Limit the first line of the commit message to 72 characters.
- Include a detailed description of your changes in the commit.
- Use the following format for commit messages:
  
  `[TYPE]: [SUBJECT]`
  
  Where `[TYPE]` can be one of the following: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`.

Example:
```
feat: add new travel package feature
```

## Pull Request Process
1. Fork the repository and create your branch from `main`.
2. Make sure your code follows the coding standards outlined above.
3. Test your changes thoroughly before submitting your pull request.
4. When ready, submit a pull request with a clear description of the changes made. 
5. Ensure your pull request references the issue it resolves, if applicable.

## Development Workflow
1. **Clone the repository**: Clone your fork and set the original repository as upstream.
2. **Create a branch**: Create a new branch for your feature or fix.
3. **Make your changes**: Implement your feature or fix any bugs.
4. **Commit your work**: Use the commit conventions outlined above.
5. **Push your changes**: Push to your fork on GitHub.
6. **Open a pull request**: Go to the original repository and create a pull request to `main`.

Thank you for considering contributing to the Destiny Tour & Travels project! Your contributions help us create a better experience for everyone.
